  1. 一些概念的简单描述

    1. HTML--结构，主要负责前端的结构，前端的展示。

      CSS---美化

    2. XML --- 可扩展的标记型语言，可以自定义标签，主要作用用来写配置文件。像solr的配置文件就是用这个写的。主要负责后台配置的事。（移动端也用于前端）

    3. Tomcat三个核心组件，以下三个统称为服务器端组件

      • servlet作用

        • 获取浏览器发送的请求

        • 处理请求

      servlet是Tomcat里最重要的组件

      • Filter 过滤数据的

      • Listener监听用户相应的操作

    4. HTTP协议：超文本传输协议

      浏览器与服务端交互的过程需要遵循相应的协议，就是HTTP协议。HTTP协议属于应用层协议

      要求传输的数据满足一些规则，比如要求传输的数据有请求头、请求行、请求体。响应的时候有响应头、响应行、响应体（响应行里面有响应码，404、200、302、500）

      TCP、UDP协议属于传输层协议

      IP属于网络层协议。

      TCP/IP模型分为物理+数据链路层、网络层、传输层、应用层。

    5. JSP---Java服务器端的页面

      servlet能给我们响应数据，servlet不适合展示数据，servlet是处理业务逻辑！让servlet负责显示的工作，做不好。

      使用JSP可以实现一个动态的页面的展示。

    6. 会话控制

      Cookie

      Session

      服务器端会不断地获取到请求，服务器端怎么判断这几个请求是否是来自同一个浏览器（客户端）呢，服务器不知道，为了让服务器端识别出来多次请求是同一个浏览器，就引入了cookie和session，服务端叫session，客户端叫cookie

    7. Ajax

      集成在JavaScript中。比如登录网站的时候，写了用户名，光标移向另一个框的时候，通常用户名框后面会有提示，这是因为此时浏览器已经发送了一个请求到服务器，再通过服务器（这里可以理解为后端）到数据库中去进行用户名的校验，我们还没有点提交或者说注册按钮，就已经发送请求了。Ajax用来实现异步请求，这种场景就是异步请求。

      同步异步是什么意思呢？

      比如同步就是一段时间内，只能有一个线程执行操作，其余的线程就只有等待，所以我们也有同步代码块和同步方法的方式来保证线程的安全。异步就是大家一起执行，虽然我们没有点击注册按钮，但是不影响输入用户名之后浏览器发送请求给服务器，这就是一个异步的场景。

      比如在百度里搜索数据，刚写了几个汉字，下面就会出现提示，也就是浏览器异步向服务器发送了请求。

  2. GET请求的特点是：

    1. 浏览器地址栏中的地址是：action属性（提交的服务器地址（协议+ip地址+端口号）） + ？ + 请求参数

      请求参数的格式是：name=value&name=value

      提交的参数，在URL的请求参数列表显示得一清二楚

    3. 提交参数有数据长度的限制

    1. 浏览器的地址栏只有action属性值，即协议+ip地址+端口号
    2. 相对于GET请求更安全
    3. 理论上没有数据长度的限制

  16. DOM模型（Document Object Model）

    • Document管理了所有的HTML文档内容
    • 它是一种树结构的文档，有层级关系
    • 它让我们把所有的标签都对象化
    • 我们可以通过document访问所有的标签对象

  17. 我们通过document的方法来获取标签对象

    要想对一个标签对象进行处理前，必须先获得这个标签对象，比如说要想查询标签对象的内容，想修改标签对象的内容，总之想对这个标签对象进行处理，必须先获得此标签对象，就通过document提供的方法。这里说的标签对象就是dom对象的意思，获得了dom对象，就可以拿到其中的属性，**这里说的属性就是此dom对象中的各个元素。**比如说文本的内容，属性的值等。

    document.getElementsByName(elementName)

    通过标签的name属性查找dom对象，elementName是标签的name属性值，返回的是dom对象集合

    document.getElementById(id)

    通过标签的id查找dom对象，id是标签的id属性值

    document.getElementByTagName(tagname)

    通过标签名查找dom对象，返回的是dom对象集合，tagname是标签名（前面的id和name是属性值，这里指的是标签名）

    document.createElement(tagName)

    创建标签对象。tagname是要创建的标签名。

    注：

    1. document对象的三个查询方法，如果有id属性，优先使用getElementById方法来进行查询

    2. 如果没有id属性，则优先使用getElementsByName来进行查询

    3. 如果id属性和name属性都没有，最后再按标签名进行查询

      因为以上三个方法从上到下，查询到的范围是越来越大的。

    4. 注意：这几个查询方法，都必须等页面加载完毕之后再执行。

  18. 整个浏览器加载的时候是从上到下加载的，浏览器读到标签才进行创建，如果浏览器读到语句，则立刻执行。

    浏览器加载完页面，会执行window.onload事件，在这里面查询，是永远不会有问题的，因为这里面的语句是等浏览器加载完再执行。

    但也不是说查询语句必须写在window.onload里面，只要等页面加载完成再执行就可以，比如把script语句写在最下面。

# 什么是Servlet

  1. Servlet是JavaEE规范之一，规范就是接口，Servlet是接口
  2. Servlet是**javaweb三大组件之一，三大组件（也可以说是Tomcat三个核心组件）**分别是：Servlet程序、Filter过滤器、Listener监听器，这三个组件统称为服务器端组件
  3. Servlet是运行在服务器上的Java小程序，它可以接收客户端发送过来的请求，并响应数据给客户端

# Servlet的作用：

  1. 获取客户端发送过来的请求
  2. 处理请求
  3. 响应请求

# 怎么实现Servlet程序

# 方式一：

  1. 编写一个类去实现Servlet接口（Servlet组件是运行在服务器端的Java小程序，是javaweb三大组件之一。）

  2. 实现接口的service方法，处理请求，并响应数据，service方法就是专门用来处理请求响应数据的

    public class HelloServlet implements Servlet {

    @Override
    public void init(ServletConfig servletConfig) throws ServletException {

    }

    @Override
    public ServletConfig getServletConfig() {
        return null;
    }

    /**
     * service方法专门用来处理请求和响应，简单来说就是只有HelloServlet程序， 就是执行这个方法
     * @param servletRequest
     * @param servletResponse
     * @throws ServletException
     * @throws IOException
     */
    @Override
    public void service(ServletRequest servletRequest, ServletResponse servletResponse) throws ServletException, IOException {
        System.out.println("Hello Servlet被访问了");
    }

    @Override
    public String getServletInfo() {
        return null;
    }

    @Override
    public void destroy() {

    }
}

  3. 到web.xml中去配置servlet程序的响应地址。

    <?xml version="1.0" encoding="UTF-8"?>
<web-app xmlns="http://xmlns.jcp.org/xml/ns/javaee"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee http://xmlns.jcp.org/xml/ns/javaee/web-app_4_0.xsd"
         version="4.0">

    <servlet>
        <!-- servlet-name 标签给Servlet程序起一个别名（一般是类名），这里是有名字了，但是还没有地址，需要servlet-mapping来配置-->
        <servlet-name>HelloServlet</servlet-name>
        <servlet-class>com.example.javawebTest1.HelloServlet</servlet-class>
    </servlet>

    <!-- servlet-mapping标签给servlet程序配置访问地址-->
    <servlet-mapping>
        <!-- servlet-name标签的作用告诉服务器，我当前配置的地址给哪个servlet程序使用-->
        <servlet-name>HelloServlet</servlet-name>
        <!-- url-pattern标签配置访问地址
         /斜杠在服务器解析的时候，表示地址为：http://ip:port/工程路径
         /hello表示在上面的已经配置好的地址之后的地址,为：http://ip:port/工程路径/hello -->
        <url-pattern>/hello</url-pattern>
    </servlet-mapping>
</web-app>

  1. 采用WebServlet注解配置的方式（注意：这种方式就不需要再在web.xml中进行配置了）

    @WebServlet(name = "helloServlet", value = "/hello-servlet")
public class HelloServlet extends HttpServlet {

    }
}

# 输入URL到访问到页面的原理：

  1. 客户端即浏览器输入URL

    http://localhost:8080/webTest/hello

    • http:// 表示http协议

    • localhost：服务器ip----通过IP地址，定位主机（硬件）

    • 8080：服务器端口号-----通过端口号，定位主机上的软件，在这里即指服务器软件，比如Tomcat

    • /webTest: 工程路径

    • /hello：资源路径（可以是index.html，这种是静态资源，也可以通过servlet来进行配置，访问web.xml中配置的url-pattern，或者通过servlet注解中的path来进行访问）

  2. 服务器ip地址是唯一的，比如说在浏览器中输入URL，其中的域名会解析成IP，IP会定位到互联网上唯一的一台主机（硬件），这一步可以理解为定位到硬件服务器

  3. 只有硬件不行，还要定位到软件，即通过端口号来定位到和哪个软件进程进行通信，定位到Tomcat软件服务器，我们默认启动Tomcat服务器（这里的服务器指的就是软件了，而不是具体的机器）

  4. Tomcat服务器里部署了很多web工程，工程路径是唯一的，通过工程路径确定访问哪个web工程

  5. 资源路径决定访问哪个资源！（这里的资源路径并不一定是xxx.html，这个路径是指能通过这个路径访问到资源）优先去web.xml配置文件里找url-pattern，根据url-pattern，以及servlet-name，找到具体的类

  6. 这个具体的类里面就有service方法，这个service方法专门执行功能代码，也就是前面说的servlet组件获取请求、处理请求、响应请求！

    public void service(request, response) {
    // 执行功能代码！
}

  7. 如果采用的是注解+继承HttpServlet的方式，那么就实现对应的方法

总结：URL----ip定位主机------端口号定位服务器------工程名定位具体的web工程-----资源路径定位资源（根据资源路径找到具体的servlet程序）-----servlet程序获取请求、处理请求、响应请求。

# servlet程序的生命周期

  1. 执行servlet构造器方法
  2. 执行init初始化方法
  3. 执行service方法
  4. 执行destroy销毁方法

说明：第一、二步是在第一次访问的时候，创建Servlet程序调用，说明Servlet的对象只允许创建一次，构造器方法只能执行一次，说明这个Servlet对象是单例的，只允许创建一个Servlet对象。

第三步，是每次访问都会调用。

第四步，是停止服务器的时候，才会执行，销毁。释放内存！

# 十、JSP

  1. 什么是JSP

    JSP的全称是Java server pages，Java的服务器页面

  2. JSP的主要作用是代替Servlet程序回传HTML页面的数据

    因为Servlet程序回传HTML页面数据是一件非常繁琐的事情，开发成本和维护成本都极高

  3. jsp页面如何访问

    jsp页面和html页面一样，都是存放在web目录下，访问也跟访问html页面一样

    比如，在web目录下有如下的文件：

    web 目录

    	a.html      访问地址是   http://ip:port/工程路径/a.html

    ​ b.jsp 访问地址是 http://ip:port/工程路径/b.jsp

  4. JSP页面的本质：

    jsp页面本质上是一个Servlet程序

    当我们第一次访问jsp页面的时候，Tomcat服务器会帮我们把jsp页面翻译成为一个java源文件，并且对它进行编译，称为.class字节码程序。我们打开java源文件，不难发现其里面的内容是一个类，并且继承了org.aoache.jasper.runtime.HttpJspBase类，这个类又继承了HttpServlet类，所以这个源文件的类间接继承了HttpServlet类。也就是说，翻译出来的是一个Servlet程序。

    HttpJspBase是所有jsp对应的servlet程序的超类或者说父类

    jsp页面对应的Servlet程序的源代码，其底层实现，也是通过输出流，把html页面数据回传给客户端

  5. Request、Response对象都是由Tomcat服务器创建，每次请求创建一次，请求结束后销毁！不需要我们自己去创建，所以重写doGet/doPost方法，直接用形参里的request对象和response对象

  6. jsp的page指令可以修改jsp页面中的一些重要的属性或者行为

    • language属性：表示jsp翻译后，是什么语言，暂时只支持Java

    • contentType属性：表示jsp返回的数据类型是什么，也是源码中response.setContentType()参数值，比如"text/html; charset=utf-8"

    • pageEncoding属性：表示当前jsp页面文件本身的字符集

    • import属性：跟Java源代码中一样，用于导包、导类

    ----------------以下两个属性是给out输出流使用-----------

    • autoFlush属性：设置当out输出流缓冲区满了之后，是否自动刷新缓冲区，默认值是true

    • buffer属性：设置缓冲区的大小，默认8KB

    • errorPage属性：表示jsp页面出错后，自动跳转去的页面路径，这个路径一般都是以斜杠打头

      它表示请求地址为：http://ip:port/工程路径/

      本来斜杠属性被服务器解析是到http://ip:port/工程路径/ ，被浏览器解析是到http://ip:port

      在服务器映射到代码的webapp目录

      因为jsp页面本质是servlet程序（在这里理解为服务器程序），所以斜杠表示http://ip:port/工程路径/

    • isErrorPage：设置当前jsp页面是否是错误信息页面，默认是false，如果是true，可以获取异常信息

    • session属性：设置访问当前jsp页面，是否会创建HttpSession对象，默认是true

    • extends属性：设置jsp翻译出来的Java类默认继承哪个类

  7. jsp可以声明脚本

    声明Java类结构脚本的格式是：

    <%! 声明Java代码 %>

    作用：可以给jsp页面翻译出来的Java类，声明属性、方法、内部类、静态代码块

    声明表达式脚本的格式是：

    <%=表达式%>

    表达式脚本的作用是 在jsp页面上 输出数据

    表达式脚本的特点：

    1. 所有的表达式脚本都会被翻译到_jspService方法中
    2. 表达式脚本都会被翻译成out.print()输出到页面上
    3. 表达式脚本是在jspService()方法中，所以可以直接使用这个方法里声明的对象
    4. 表达式脚本不能用分号结束
    <%
	java语句
%>

    代码脚本的作用是:可以在jsp页面中，编写我们自己需要的功能（写的是Java语句）

    代码脚本的特点：

    1. 代码脚本翻译之后都在_jspService方法中
    2. 代码脚本由于翻译到_jspService方法中，所以在_jspService方法的现有对象可以直接使用
    3. 代码脚本还可以由多个代码脚本块组合完成一个完整的Java语句
    4. 代码脚本还可以和表达式脚本一起组合使用，在jsp页面上输出数据

    • out---- jsp输出流对象
    • exception ---- 异常对象

  10. jsp四大域对象

    • PageContext（PageContextImpl类） 当前jsp页面范围内有效
    • Request（HttpServletRequest类） 一次请求内有效
    • Session（HttpSession类） 一个会话范围内有效（打开浏览器访问服务器，直到关闭浏览器）
    • application（ServletContext类） （整个web工程范围内都有效，只要web工程不停止，数据都在）

    域对象是可以像Map一样存取数据的对象，四个域对象功能一样，不同的是他们对数据的存取范围

    虽然四个域对象都可以存取数据，但是在使用上它们有优先顺序：

    pageContext --- Request---- Session --- application

    适用范围从小到大，这是对内存的优化，使存储在内存中的数据得到最快的释放

  11. jsp中的out输出和response.getWriter()输出的区别

    response中表示响应，我们经常用于设置返回给客户端的内容（输出）

    out也是给用户做输出使用的

    2. 会执行response的刷新操作，把全部数据写给客户端（浏览器）

     @Override
    protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {

        //首先判断上传的数据是否是多段数据（我这里判断不了，需要适配Tomcat10的commons-fileupload jar包或者把Tomcat版本降低）
        // 然后执行下面的操作
        FileItemFactory fileItemFactory = new DiskFileItemFactory();
        ServletFileUpload servletFileUpload = new ServletFileUpload(fileItemFactory);
        try {
            List<FileItem> fileItems = servletFileUpload.parseRequest((javax.servlet.http.HttpServletRequest) req);
            for (FileItem fileItem: fileItems) {
                if (fileItem.isFormField()) {
                    System.out.println("name属性值：" + fileItem.getFieldName());
                    System.out.println("value属性值：" + fileItem.getString("UTF-8"));
                } else {
                    System.out.println("name属性值：" + fileItem.getFieldName());
                    System.out.println("文件名：" + fileItem.getName());
                    fileItem.write(new File(fileItem.getName()));
                }
            }
        } catch (FileUploadException e) {
            e.printStackTrace();
        } catch (Exception e) {
            e.printStackTrace();
        }
    }

        String mimeType = servletContext.getMimeType("/static/img/" + downloadFileName);
